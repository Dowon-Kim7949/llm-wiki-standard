# 팀 발표 자료 (Team Briefing) — 관리 안내

전 인원 대상 **llm-wiki-governance 소개 발표** 자료 모음입니다. 발표용 슬라이드(덱)와 발표자 노트를 **한 세트로 버전 관리**합니다.

## 구성

| 파일 | 용도 |
|---|---|
| [`llm-wiki-briefing.html`](./llm-wiki-briefing.html) | 발표 슬라이드(덱). 방향키 `←`/`→`로 넘김. 브라우저 인쇄(`Ctrl+P`)로 슬라이드당 1페이지 PDF 생성 가능. |
| [`SPEAKER_NOTES.md`](./SPEAKER_NOTES.md) | 슬라이드별 발표자 스크립트(시간 배분 포함). |
| `README.md` | 이 파일 — 관리 규칙. |

- **온라인 공유용 Artifact 링크**: https://claude.ai/code/artifact/13dfc476-93ab-4d6d-a1c5-fbaec82135a6
  (덱과 동일 내용을 게시한 것. 위 HTML을 고치면 재게시하여 갱신한다.)
- **대상 버전**: `v1.22.0` (덱 표지·마무리와 노트 상단 표기가 기준. `package.json` 버전과 맞춘다. 22장.)

> 이 폴더(`outputs/`)는 `package.json` `files`에 없으므로 **npm 패키지에는 포함되지 않습니다**(사내 자료).

## 관리 규칙 (핵심)

1. **기능이 바뀌면 발표 자료도 같이 바꾼다.** 사용자 체감(명령·옵션·동작·사용법)이 바뀌는 변경은, 관련 **슬라이드(HTML)와 발표자 노트(MD)를 함께** 갱신한다.
2. **배포 전에 갱신한다.** 릴리스 준비 시 [`RELEASE_CHECKLIST.md`](../../RELEASE_CHECKLIST.md)의 "Team Materials" 항목에서 확인한다.
3. **버전 표기를 맞춘다.** 덱 표지(`v1.0.0 → vX.Y.Z`)·마무리(`감사합니다 · vX.Y.Z`)·노트 상단의 "대상 버전"을 새 `package.json` 버전으로 갱신한다.
4. **슬라이드와 노트는 1:1로 유지한다.** 슬라이드를 추가/삭제/재배치하면 `SPEAKER_NOTES.md`의 번호와 순서도 맞추고, 덱 하단 카운터 총 장수(`NN / 총장수`)도 확인한다.
5. **온라인 링크는 재게시로 갱신한다.** HTML을 고친 뒤 같은 Artifact URL로 다시 게시하여 링크를 유지한다.

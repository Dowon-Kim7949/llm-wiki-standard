## 요약 · Summary

- 에이전트 네이티브(MCP). 에이전트가 CLI를 spawn하지 않고 위키를 툴로 질의·점검하게 한다. 하위호환 — 새 명령·모듈만 추가하며 CLI·JSON·프로그래매틱 API·frontmatter 계약은 불변. · Agent-native (MCP): agents query and check the wiki as tools instead of shelling out. Backward-compatible — a new command and module only.

## 추가 · Added

- `llm-wiki mcp` — stdio 위의 Model Context Protocol 서버(개행 구분 JSON-RPC 2.0). Node 내장만으로 구현(서드파티 MCP SDK 없음)해 무의존성 정책 유지. `{ "command": "npx", "args": ["-y", "@dowonk-7949/llm-wiki-standard", "mcp"] }`로 MCP 클라이언트에 등록. · a Model Context Protocol server over stdio, hand-rolled on Node built-ins (no SDK).
- 읽기 전용 MCP 툴 10개: `validate`·`audit`·`next`·`status`·`doctor`·`stats`·`graph`·`explain`·`handoff`·`prompt`. 쓰기/변경 명령 미노출, 어떤 툴도 파일을 쓰지 않음(`readOnlyHint`). 각 `tools/call`은 명령 결과(`schemaVersion` 포함)를 `structuredContent`로, 사람용 요약을 텍스트로 반환. 예외는 `isError`. · ten read-only MCP tools; no write command exposed; results as structuredContent + text.
- 패키지 진입점의 프로그래매틱 MCP 표면: `startMcpServer`·`MCP_TOOLS`·`handleMcpMessage`·`MCP_PROTOCOL_VERSION`. · programmatic MCP surface from the package entry point.

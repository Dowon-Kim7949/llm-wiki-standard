## 수정 · Fixed

- **비-UTF-8 매니페스트 오분류 교정.** UTF-16/UTF-8-BOM으로 저장된 매니페스트(예: Windows에서 저장된 `requirements.txt`)가 UTF-8로 읽혀 mojibake가 되면서 프레임워크 키워드를 놓쳐 FastAPI 백엔드가 `library`로 오분류됐다. detector의 매니페스트/소스 읽기를 BOM 인식 리더(`readTextAuto`/`decodeWithBom`)로 교체해 UTF-16LE·UTF-16BE·UTF-8 BOM을 올바르게 디코드한다. BOM 없는 파일은 byte-identical, 위키 문서 인코딩 스캔은 불변. · A manifest saved as UTF-16/UTF-8-BOM was read as UTF-8 and mis-typed a FastAPI backend as `library`. A BOM-aware reader now backs the detector's manifest/source reads; BOM-less files are byte-identical and the wiki-doc encoding scan is unchanged.
- **handoff 프롬프트가 미생성 어댑터 파일을 가리키던 문제 교정.** `--agent` 미지정 시 `quickstart`/`init`은 어댑터 파일을 만들지 않는데 프롬프트는 존재하지 않는 `AGENTS.md`/`CLAUDE.md`를 먼저 읽으라고 시작했다. 이제 명시적으로 선택된 에이전트의 파일만 진입점에 넣고 기본값은 `docs/llm-wiki/index.md`다. · The handoff prompt no longer opens by pointing at an `AGENTS.md`/`CLAUDE.md` that setup never created; it names adapter files only for explicitly-selected agents and otherwise anchors on the wiki index.

## 변경 · Changed

- **모드 플래그 없는 `init`/`quickstart`이 안내로 읽힌다.** `--dry-run`/`--write` 둘 다 없으면 이전에는 `Blocked`(exit 2)로 실패처럼 보였다. 이제 `Ready (needs --write)` + `Next Step`을 렌더하고 exit 0으로 끝난다(`next`의 `ready` 결과와 동일). `--dry-run`+`--write` 동시 지정은 여전히 거부. · Running `init`/`quickstart` with no mode flag now renders `Ready (needs --write)` and exits 0 instead of a `Blocked` report; requesting both modes at once is still rejected.
- **handoff `Next Step`이 워크플로를 설명한다(온램프 명료화).** `Handoff Prompt`는 CLI가 실행하는 게 아니라 저장소에서 연 코딩 에이전트(Claude Code/Codex)에 붙여넣는 지시문 → 에이전트가 코드 근거로 문서(도메인별 `domains/*.md` 포함) 보강 → 사람이 검토해 `verified`, 라는 3단계를 명시한다. · The handoff `Next Step` now spells out that the prompt is pasted into a coding agent (not run by the CLI), which fills in the docs for a human to review.
- **`quickstart` 출력이 브라운필드를 인식한다.** skip 개수에 사유 주석(`skipped: N (M already exist, kept)`), 새로 만들 문서가 없으면 기존 문서를 handoff로 보강하라는 안내를 덧붙인다(“도구가 아무것도 안 한 것”처럼 보이지 않게). · `quickstart` annotates the skipped count and, on an existing wiki, points to enriching the existing docs rather than reading as a no-op.
